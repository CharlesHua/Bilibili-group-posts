# Bilibili 分组动态扩展

这是一个基于 Chrome Manifest V3 的浏览器扩展，用来把 Bilibili 的关注分组同步到动态页中，并在扩展弹窗里提供更方便的分组管理能力。

当前版本：`1.1`

## 当前已支持的主要功能

- 同步当前账号的关注分组和分组成员
- 在动态页插入自定义分组标签
- 按分组过滤动态
- 在弹窗或网页模式中查看所有分组及成员
- 获取某个分组或全部关注对象的“最后动态更新时间”
- “获取最后更新”支持 `全部` / `补全` 两种模式
- 边获取边显示最后更新时间
- 记录每个按钮的“上次获取时间”
- 在刷新分组时复用旧缓存中的用户附加信息，例如最后更新时间
- 提供设置页面，可控制时间显示格式

## 项目目标

原始的 Bilibili 动态页并不直接支持“按关注分组筛选动态”。

这个扩展的目标是：

1. 从 Bilibili 接口读取当前账号的关注分组
2. 将分组显示到动态页导航区域
3. 点击分组后，只显示该分组下 UP 主的动态
4. 在扩展界面中更方便地查看分组成员、同步状态和最后更新时间

## 核心功能说明

### 1. 自动识别当前登录账号

扩展会请求：

- `https://api.bilibili.com/x/web-interface/nav`

用于获取：

- 当前用户 `mid`
- 当前用户名 `uname`

这些信息会缓存到 `chrome.storage.local`，并显示在弹窗顶部。

### 2. 同步关注分组和成员

扩展会请求：

- `https://api.bilibili.com/x/relation/tags?vmid={mid}`
- `https://api.bilibili.com/x/relation/tag?tagid={tagId}&pn={page}`

同步完成后会在本地保存：

- 分组列表 `tags`
- 每个分组下的用户数组 `tag_{tagId}`

### 3. 动态页插入分组标签

在 `https://t.bilibili.com/` 页面中，内容脚本会等待原始动态导航加载完成，然后插入扩展自己的分组标签。

目前支持：

- 只显示在弹窗中勾选启用的分组
- 标签过多时自动换行到下一行导航栏
- 点击标签后按该分组过滤动态

### 4. 按分组过滤动态

过滤逻辑位于 `content.js`，当前做法是：

- 读取选中分组下的成员列表
- 提取成员的 `uname`
- 在当前已加载的动态卡片中查找作者显示名
- 显示命中的动态，隐藏未命中的动态

说明：

- 当前是基于作者显示名匹配，不是基于更稳定的 uid
- 如果 Bilibili 页面结构或作者显示方式变化，过滤逻辑可能需要调整

### 5. 弹窗界面和网页模式

点击扩展图标后，可以看到：

- 当前账号 MID 和用户名
- 所有已同步的关注分组
- 每个分组的人数
- 每个分组下的成员列表
- 是否显示在动态页中的勾选状态
- 折叠/展开某个分组成员列表

同时支持两种显示方式：

- 小窗模式
- 网页模式

在页头右侧可以：

- 点击 `⚙` 进入设置页面
- 点击 `网页显示` 切换到完整网页模式
- 在网页模式中点击 `切回小窗`

说明：

- 网页模式与小窗模式使用同一套 `popup.html + popup.js`
- 只是通过 URL 参数切换布局和页面模式

### 6. 设置页面

点击页头右上角的齿轮按钮 `⚙` 可以打开设置页面。

当前设置项：

- `显示设置`
  - `显示时间`
    - `显示 YYYY-MM-DD`
    - `简短显示`

简短显示规则：

- 如果年份是今年，则只显示 `MM-DD`
- 如果不是今年，则仍显示 `YYYY-MM-DD`

设置页底部提供：

- `应用`
- `取消`

点击 `应用` 后，新的显示格式会写入 `chrome.storage.local`，并立即作用到当前已显示的成员最后更新时间。

### 7. 获取最后动态更新时间

扩展支持两种范围：

- 顶部“获取最后更新”：更新所有关注对象
- 每个分组里的“获取最后更新”：只更新该分组

并且每个“获取最后更新”都支持两种模式：

- `全部`
  - 更新当前范围内的全部关注对象
- `补全`
  - 只更新当前范围内还没有 `latestUpdateDate` 的关注对象

更新时间来自接口：

- `https://api.bilibili.com/x/polymer/web-dynamic/v1/feed/space?host_mid={uid}`

处理规则：

- 读取该 UP 的动态列表
- 跳过带“置顶”标记的动态
- 取第一条正常动态的时间
- 格式化为 `YYYY-MM-DD`

### 8. 边获取边显示

获取最后更新时间时，不需要等整个任务全部结束。

当前实现是：

- 每拿到一个用户的最新时间
- 就立即发送消息给 popup
- popup 直接更新对应这一行的显示

所以你会看到日期一条条出现。

### 9. 低频访问策略

为了避免短时间高频访问同一接口导致被限制，当前实现采用了：

- 小并发队列，默认并发数 `2`
- 每次请求前增加随机延迟，范围 `600ms ~ 1500ms`

这套策略只用于“获取最后更新时间”，不会影响普通分组同步。

### 10. 保留旧缓存中的用户附加信息

点击“手动更新分组”时，扩展不会直接粗暴丢掉旧数据。

当前做法：

1. 先读取旧的分组用户数据
2. 将所有旧分组用户按 `uid(mid)` 合并成临时缓存
3. 获取新的分组和分组成员
4. 对于新分组里的每个用户，按 uid 在旧缓存中查找同一用户
5. 把旧用户对象中的附加信息迁移到新数据里

目前主要用于保留：

- `latestUpdateDate`

这样即使某个用户换了分组，也不会因为“手动更新分组”而丢掉之前已经获取到的最后更新时间。

### 11. 显示“上次获取时间”

扩展会记录：

- 顶部“获取最后更新”按钮上次成功执行的时间
- 每个分组按钮上次成功执行的时间

显示格式：

- `上次获取时间：YYYY-MM-DD HH:MM`

这些时间保存在 `chrome.storage.local` 中，重新打开弹窗仍会保留。

## 项目结构

```text
.
├── manifest.json   # 扩展清单，声明权限、注入页面、弹窗、后台脚本
├── background.js   # 后台 service worker，负责同步、缓存、更新时间获取
├── content.js      # 内容脚本，负责动态页插入标签和过滤动态
├── popup.html      # 弹窗 / 网页模式 / 设置页共用的界面结构与样式
├── popup.js        # 弹窗 / 网页模式 / 设置页的交互逻辑
├── icon.ico        # 扩展图标
└── README.md       # 项目说明
```

## 运行机制

### background.js

主要职责：

- 获取当前登录用户信息
- 同步关注分组和分组成员
- 将结果写入 `chrome.storage.local`
- 接收 popup 发来的“手动更新分组”“获取最后更新时间”请求
- 控制最后更新时间获取时的小并发和随机延迟
- 在刷新分组时把旧缓存中的附加信息迁移到新数据

### content.js

主要职责：

- 监听页面 DOM 变化
- 检测动态页导航栏何时出现
- 插入扩展自己的分组标签
- 点击分组后过滤当前动态列表
- 页面懒加载更多动态时继续应用过滤规则

### popup.js

主要职责：

- 渲染分组列表和成员列表
- 展示当前用户信息
- 处理网页模式 / 小窗模式切换
- 打开设置页面
- 保存与应用显示设置
- 发起手动更新分组
- 发起全量或按组获取最后更新时间
- 处理“全部 / 补全”两种更新模式
- 实时接收后台进度消息和单个用户的更新时间消息
- 显示“上次获取时间”

## 本地存储结构

项目主要使用 `chrome.storage.local`。

### 基础字段

- `mid`: 当前登录用户 uid
- `username`: 当前登录用户名
- `tags`: 分组列表
- `tag_{tagId}`: 某个分组下的用户数组
- `groupStates`: popup 中每个分组的 UI 状态
- `latestDateFetchMeta`: 上次获取最后更新时间的时间记录
- `displaySettings`: 设置页面中的显示配置

### `groupStates` 示例

```json
{
  "12345": {
    "checked": true,
    "collapsed": false
  }
}
```

### 用户对象常见字段

```json
{
  "mid": 123456,
  "uname": "示例用户",
  "latestUpdateDate": "2026-02-26"
}
```

### `latestDateFetchMeta` 示例

```json
{
  "all": "2026-03-20 16:48",
  "groups": {
    "12345": "2026-03-20 16:52"
  }
}
```

### `displaySettings` 示例

```json
{
  "dateDisplayMode": "short"
}
```

## 安装方式

### 1. 下载代码

```bash
git clone https://github.com/CharlesHua/Bilibili-group-posts.git
```

或者直接下载 ZIP 后解压。

### 2. 加载扩展

1. 打开 Chrome 浏览器
2. 访问 `chrome://extensions/`
3. 打开右上角“开发者模式”
4. 点击“加载已解压的扩展程序”
5. 选择当前项目目录

## 使用流程

### 首次使用

1. 在浏览器中登录 Bilibili
2. 打开扩展弹窗
3. 点击“手动更新分组”
4. 等待分组和成员同步完成
5. 勾选希望显示在动态页中的分组

### 动态页过滤

1. 打开 `https://t.bilibili.com/`
2. 找到扩展插入的自定义分组标签
3. 点击某个分组
4. 查看该分组对应的动态

### 获取最后更新时间

1. 在 popup 中点击“获取最后更新”
2. 选择：
   - `全部`
   - `补全`
3. 观察按钮上的 `更新中...（x/y）`
4. 观察成员列表里的日期一条条出现

### 调整时间显示格式

1. 点击页头右上角 `⚙`
2. 打开设置页面
3. 在“显示设置 -> 显示时间”中选择格式
4. 点击“应用”

## 已知限制

### 1. 动态过滤基于作者显示名

当前不是按 uid 过滤，而是按 `uname` 与页面上的作者显示名匹配。

风险：

- 页面显示名与接口返回值不一致时，可能过滤错误
- 作者改名后可能需要重新同步数据

### 2. 依赖 Bilibili 页面 DOM 结构

内容脚本依赖这些类名：

- `.bili-dyn-list-tabs`
- `.bili-dyn-item`
- `.bili-dyn-title__text`

如果 Bilibili 改版，这部分逻辑可能失效。

### 3. “切回小窗”受浏览器限制

网页模式可以关闭自身，但不能强制直接弹出扩展小窗。

所以“切回小窗”本质上是关闭当前页，回到扩展的常规使用方式。

### 4. 最后更新时间仍可能受接口变化影响

虽然当前已经处理了：

- 跳过置顶动态
- 兼容多个常见时间字段

但如果 Bilibili 修改了动态接口结构，仍可能需要调整解析逻辑。

## 推荐的后续优化方向

- 让动态页切换分组时不再刷新页面
- 把动态过滤从 `uname` 升级为更稳定的标识方式
- 增加分组搜索和成员搜索
- 增加错误重试与退避策略
- 给刚更新出的日期做轻量高亮效果
- 在网页模式下做更适合长列表管理的布局

## 调试建议

### 查看后台日志

1. 打开 `chrome://extensions/`
2. 找到本扩展
3. 进入 Service Worker 调试界面
4. 查看 `background.js` 的日志输出

### 查看内容脚本日志

1. 打开 Bilibili 动态页
2. 按 `F12`
3. 在页面 DevTools Console 中查看 `content.js` 输出

### 查看 popup 日志

1. 打开扩展弹窗
2. 对 popup 单独打开开发者工具
3. 查看 `popup.js` 输出
